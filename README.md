*Выводы:*

**Качество:** 

* Качество растет с ростом эпохи, для обеих моделей наибольшее качество достигается без использования ddp и с меньшим из размеров батчей.
* Для маленькой модели четко видно, что при одинаковом размере батча и одинаковом номере эпохи большее качество у модели с меньшим число ddp процессов.
* Для большой модели уже не наблюдается такая закономерность, но видно, что когда использовали 4 ddp процесса, то качество было всегда хуже по сравнению с другими двумя запусками.
* Также для маленькой модели видим, что качество для запусков с 4 ddp процессами, размером батча 64 и с 2 ddp процессами, размером батча 128 ведет себя почти одинаково, то же самое верно для пары запусков с 4 ddp процессами, размером батча 128 и с 2 ddp процессами, размером батча 256.

**Время работы:**

Проход по батчу:
* Для маленькой и большой модели время затрачиваемое на проход по батчу растет с увеличением количества ddp процессов для любого размера батча. Но в случае маленькой чем больше размер батча, тем сильнее различия(сначала видим как будто линейный рост, затем ближе к квадратичному), а в случае большой рост уже не такой большой и чем больше размер батча, тем рост слабее.

Время на эпоху:
* Для маленькой модели время затрачиваемое на эпоху растет с увеличением количества ddp процессов, то есть время на синхронизацию не компенсируется сэкономленным засчет распараллеливания. В целом можно сказать что рост примерно одинаковый для всех размеров батча.
* Для большой модели видим обратную закономерность - время затрачиваемое на эпоху падает с увеличением количества ddp процессов. Здесь тоже можно сказать, что рост примерно одинаковый для всех размеров батча.
